## 📋 Описание проекта

**GitHub User Finder** — это десктопное GUI-приложение на Python для поиска и сохранения пользователей GitHub. Приложение предоставляет удобный интерфейс для работы с GitHub API, позволяя быстро находить информацию о разработчиках и сохранять интересных пользователей в избранное.

### Основные возможности

- 🔍 Поиск пользователей GitHub по логину
- 📊 Просмотр детальной информации (репозитории, подписчики, дата регистрации и т.д.)
- ⭐ Добавление пользователей в избранное
- 💾 Автоматическое сохранение избранного в JSON файл
- 🖱️ Контекстное меню (ПКМ) для быстрых действий
- 🌐 Открытие профиля в браузере
- 📁 Отслеживание пути к файлу сохранения

## 🚀 Установка

### Требования

- Python 3.7 или выше
- Библиотека `requests`

### Шаги установки

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/El1xirq/github-user-finder.git
cd github-user-finder
```

2. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

или вручную:
```bash
pip install requests
```

3. **Запустите приложение:**
```bash
python main.py
```

## 📖 Как использовать

### Поиск пользователя

1. Введите логин пользователя GitHub в поле поиска
2. Нажмите кнопку **"🔍 Найти"** или клавишу **Enter**
3. Результат отобразится в таблице с информацией:
   - Username (логин)
   - Количество репозиториев
   - Количество подписчиков
   - Кнопка действия

### Просмотр детальной информации

- **Двойной клик** по любому пользователю в результатах поиска
- **Двойной клик** по пользователю в списке избранного
- Откроется окно с подробной информацией:
  - Полное имя
  - Компания
  - Локация
  - Email
  - Количество репозиториев и подписчиков
  - Даты создания и обновления профиля
  - Ссылка на GitHub профиль

### Управление избранным

**Добавление в избранное:**
1. Найдите пользователя
2. Дважды кликните по нему
3. В открывшемся окне нажмите **"⭐ Добавить в избранное"**

**Просмотр избранного:**
- Перейдите на вкладку **"⭐ Избранные пользователи"**
- Здесь отображаются все сохранённые пользователи с датой добавления

**Удаление из избранного:**
- Через контекстное меню: ПКМ → "Удалить из избранного"
- Через окно деталей: Откройте пользователя → "❌ Удалить из избранного"
- Через кнопку: Выберите пользователя → "❌ Удалить из избранного"

### Контекстное меню (ПКМ)

Правой кнопкой мыши по пользователю доступны быстрые действия:
- Добавить/удалить из избранного
- Показать детальную информацию

### Полезные кнопки

- **"📍 Показать путь к JSON"** — показывает точное местоположение файла с избранным
- **"⭐ Избранное"** — переключение на вкладку с сохранёнными пользователями

## 🔧 Как работает GitHub API

### Используемый эндпоинт

Приложение использует **GitHub REST API v3**:

```
GET https://api.github.com/users/{username}
```

### Пример запроса

```python
import requests

response = requests.get("https://api.github.com/users/octocat")
user_data = response.json()
```

### Пример ответа API

```json
{
  "login": "octocat",
  "id": 1,
  "avatar_url": "https://github.com/images/error/octocat_happy.gif",
  "name": "The Octocat",
  "company": "GitHub",
  "blog": "https://github.com/blog",
  "location": "San Francisco",
  "email": "octocat@github.com",
  "bio": "GitHub mascot",
  "public_repos": 8,
  "followers": 10,
  "following": 5,
  "created_at": "2011-01-25T18:44:36Z",
  "updated_at": "2024-01-01T00:00:00Z",
  "html_url": "https://github.com/octocat"
}
```

### Обработка ошибок API

| Код ошибки | Значение | Действие приложения |
|------------|----------|---------------------|
| 200 | Успех | Отображение данных пользователя |
| 404 | Пользователь не найден | Сообщение об ошибке |
| 403 | Превышен лимит запросов | Сообщение о лимите |
| Сетевая ошибка | Нет подключения | Сообщение об ошибке сети |

### Лимиты API

- **Без аутентификации:** 60 запросов в час на IP-адрес
- **С аутентификацией:** 5000 запросов в час

*Примечание: Для увеличения лимита можно добавить токен авторизации (в текущей версии не реализовано)*

## ❗ Возможные проблемы и решения

### Проблема: Не удаётся найти файл favorites.json

**Решение:** Нажмите кнопку **"📍 Показать путь к JSON"** в приложении

### Проблема: Ошибка "No module named 'requests'"

**Решение:**
```bash
pip install requests
```

### Проблема: Приложение не сохраняет избранное

**Решение:**
- Проверьте права на запись в папке
- Запустите приложение с правами администратора
- Проверьте консоль на наличие ошибок

### Проблема: Превышен лимит GitHub API

**Решение:** Подождите час до сброса лимита

### Проблема: Пустое окно после поиска

**Решение:**
- Проверьте подключение к интернету
- Убедитесь, что логин введён правильно (без пробелов)
